```
3.1 记录集的叠加
问题：要将来自多个表的数据组织到一起，就像将一个结果集叠加到另一个上面一样。
UNION ALL # 要求 所有 SELECT 列表中的项目数和对应项目的数据类型必须要匹配。
UNION ALL 将包括重复的项目，如果要筛选掉重复项， 去掉 ALL。
可以使用 DSITINCT 进行去重
```

```
3.2 组合相关的行
问题：多个表有一些相同列，或有些列的值相同，要通过连接这些列得到结果。
使用 WHERE a.id = b.id (等值联接) 或使用 JOIN ON
```

```
3.3 在两个表中查找共同行
a. 等值联接
b. JOIN
c. INTERSECT
```

```
3.4 从一个表中查找另一个表没有的值
 EXCEPT (DB2 PostgreSQL)
 MINUS  (Oracle)
 子查询 (MySQL、 SQL Server)
```

```
3.5 在一个表中查找与其他表不匹配的记录
问题：对于具有相同关键字的两个表，要在一个表中查找与另外一个表中不匹配的行。
解决方案：首先返回一个表中的所有行，以及另一个表中与公共列匹配的行或不在匹配行，然后，仅保留不匹配的行。
LEFT JOIN
```

```
3.6 向查询中增加联接而不影响其他联接
问题：已经有了一个查询可以返回锁需要的值，还需要得到其他的信息，但当加入这些信息时，发现原始结果集中的数据有丢失。
先 INNER JOIN 然后 LEFT JOIN(不会丢失原始查询中的数据)；
也可以使用子查询；
```

```
3.7 检测两个表中是否有系统的数据
问题：要知道两个表或视图中是否有相同的数据（基数和值）。
MINUS EXCEPT（差集）
```

```
3.8 识别和消除笛卡尔积
FROM 子句中进行联接
```

```
3.9 聚集与联接
问题：要在包含多个表的查询中执行聚集运算，要确保表间联接不能使聚集运算发生错误。
a.DISTINCT
b.联接前先执行聚集操作
```

```
3.10 聚集与外联接
a. 外连接 && 聚集
b. SUM OVER
```

```
3.11 从多个表中返回丢失的数据
使用基于公共值的完全外连接来返回两个表中丢失的数据
```

```
3.22 在运算和比较时使用NULL值
使用 COALESCE 函数将 NULL 值转换为一个可以用来作为标准值进行比较的真实值
```
